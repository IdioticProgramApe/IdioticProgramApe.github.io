## Profiling & Applying GPU Optimizations

| Subject          | Debug Methods                                                | Potential Solutions                                          |
| ---------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| Overview         | - cmd: `stat gpu`                                            |                                                              |
| Lighting         | - light complexity viewmode<br />- cmd: `r.megalights.allowed 0/1` to enable/disable megalights<br />- cmd: `stat drawcount` to check light amount<br />- `lights` in stat gpu: light projection cost<br />- `shadow depth` in stat gpu: shadow cast cost | - do we need this amount of lights ?<br />- uncheck `Affects World` property of lights<br />- set properly `attenuation radius` & `cone angles` <br />- setup a draw distance to cull lights:<br />  - `max draw distance`: to determine whether to render<br />  - `max distance fade range`: to eliminate pop-ups<br />- set a proper `detail mode` |
| VSM              | - vsm cached pages viewmode<br />- cmd: `r.shadow.virtual.showstats 1`, check cached stats<br />  - make sure `r.shaderprint` is enabled | - further check WPO<br />                                    |
| WPO              | - nanite WPO viewmode                                        | - uncheck `Evaluate World Position offset` for static meshes<br />- contact shadows can be causing WPO being evaluated (in the Materials)<br />- properly setup the evaluation distance (only evaluate at close) |
| Particle         | - `Shift + /`, reset all instances in the world<br />- cmd: `stat niagarasystemcounts`: to check ns instance info<br />- cmd: `stat niagarasystems`: to check ns cost<br />- cmd: `stat niagara`: to get more info | - culling problems<br />  - no `Effect Type` configured in the ns (ref: `UNiagaraEffectType`)<br />    - setup a considerate `Update Frequency`<br />    - `System Scalability Settings`: `Visibility Culling`<br />  - individual culling in each render <br />  - GPU or CPU particles ? (check the bottleneck) |
| Assets           | - Tools -> Audit -> Statistics                               | - check the assets which only have one instance in the world<br />  - do we want them ? can be deleted ?<br />- check the sizes<br />  - enable nanite on hi-poly can reduce sizes |
| Static Mesh      | - Static Mesh Editor                                         | - check the triangle count, should nanite enabled or disabled ?<br />  - if nanite enabled, check fall back triangles -> subject nanite |
| Nanite           | - cmd: `nanitestats`: check shading bins stats (draw calls)<br />  - cmd: `r.nanite.showstats 0`: to disable the stats view<br />- cmd: `showflag.nanitemeshes` to view all non-nanite meshes<br />  - alternative: nanite mask viewmode <br />- nanite triangle visualization viewmode | - can be impacted by the Asset audit, if unique instances get removed<br />- keep the usage of `Material Instance Dynamic` to a moderate level<br />  - alternative: `Custom Primitive Data` (doesn't require unique materials ?) |
| Collision        | - visibility collision viewmode<br />  - might need to hide meshes to have a better view | - do we need a collision with many details ?                 |
| Scalability      | - A/B test comparison:<br />  - start cmd: `abtest <cvar> val1 val2`<br />  - stop cmd: `abtest stop` | - do we really need to turn on certain settings or give certain settings a high value ? i.e. `lensflare` |
| Sharpening       | - cmd: `r.tonemapper.sharpen <val>`                          |                                                              |
| Nanite VisBuffer | - check nanite visbuffer in stat gpu                         | - cmd: `r.nanite.maxpixelsperedge <val>` use a higher value to scale down |
